# 01. 감사 마스터 체크리스트 (Master Checklist)

본 체크리스트는 16개 핵심 영역을 기준으로 FHDL 명세서의 품질을 판정합니다.

| 영역 ID | 감사 영역 | 핵심 체크 포인트 |
| :--- | :--- | :--- |
| **A01** | 목적 및 범위 | 책임 경계(Scope In/Out)가 명확한가? |
| **A02** | 용어 및 명명 | 동의어/혼용어가 제거되고 용어 사전이 확고한가? |
| **A03** | 목표 및 비목표 | 해결하지 않는 문제(Non-Goals)가 정의되었는가? |
| **A04** | 기능 요구사항 | 입력-처리-출력이 조건문 형태로 명확한가? |
| **A05** | 입출력 계약 | 자료형, 필수여부, 에러 형식이 표준화되었는가? |
| **A06** | 데이터 모델 | 엔티티 간 관계, 제약조건, Nullable 여부가 있는가? |
| **A07** | 상태 및 흐름 | 상태 전이도(FSM)와 예외 상태가 정의되었는가? |
| **A08** | 오류 처리 | 실패 시나리오(Retry, Rollback)가 구체적인가? |
| **A09** | 비기능 요구 | 성능, 가용성, 관측성이 수치화되었는가? |
| **A10** | 보안 및 권한 | 역할 기반 권한(RBAC)과 민감 데이터 보호가 있는가? |
| **A11** | 외부 연동 | 외부 라이브러리/시스템 실패 시 Fallback이 있는가? |
| **A12** | 동시성/중복방지 | 멱등성(Idempotency)과 Race Condition 방지가 있는가? |
| **A13** | 테스트 가능성 | 명세서만으로 테스트 시나리오 유도가 가능한가? |
| **A14** | 운영 및 복구 | 배포, 설정 관리, 헬스체크 포인트가 있는가? |
| **A15** | 변경 이력 | 요구사항-설계-구현 간 추적성이 유지되는가? |
| **A16** | 설명 가능성 | 결과의 원인을 역추적할 수 있는 근거가 제시되는가? |
