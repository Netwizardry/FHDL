# 07. 비기능 요구사항 (Non-Functional Requirements)

**Status:** Active | **Version:** v4.0 | **Last Revised:** 2026-04-02 | **Supersedes:** v1.5


FHDL 시스템은 정확한 수치 계산뿐만 아니라, 엔지니어링 설계 도구로서의 안정성과 신뢰성을 보장하기 위해 다음과 같은 비기능적 품질 기준을 준수해야 합니다.

## 2. 안전 우선성 (Fail-Safe)

불확실한 상황에서 위험한 자동 결정을 내리기보다 보수적으로 멈추거나 경고하는 것을 원칙으로 한다.
*   **[R-NFR-001] 보수적 산정 원칙:** 계산 근거(필수 입력 등)가 부족할 경우 억측에 의한 자동 산정을 수행하지 않고 즉시 계산 불능 사유를 고지해야 한다.
*   **[R-NFR-002] 임의 선택 금지:** 허용 범위를 만족하는 표준 관경이 없을 경우, 임의의 관경을 추천하지 않고 **`CAL005` (Sizing Failed)** 오류를 발생시켜 설계자의 개입을 유도해야 한다.
*   **[R-NFR-003] 제약 준수 강제:** 사용자가 직접 지정한 값(`manual`)이 설계 제약조건(과속 등)을 위반할 경우, 이를 확정 결과가 아닌 '부적합' 상태로 명시하고 **`WRN001` (Velocity Violation)** 경고를 발생시켜야 한다.

## 3. 설명 가능성 (Explainability)

시스템이 도출한 모든 자동 산정 결과에 대해 설계자가 그 근거를 추적하고 검토할 수 있어야 한다.
*   **[R-NFR-004] 자동 산정 근거 제시:** 특정 관경이 선정된 이유(예: 유속 1.8m/s 만족), 총 요구양정이 결정된 이유(예: 최불리 경로 n8번 수두 합산)를 명확히 제시해야 한다.
*   **[R-NFR-005] 중간 계산 과정 분해 (Breakdown):** 최종 수치만 보여주지 않고, `마찰손실 + 정수두 + 말단요구압`과 같은 중간 계산 과정의 분해 내역(Provenance)을 제공해야 한다.
*   **[R-NFR-006] 적용 기본값 명시:** 시스템이 엔지니어링 표준 기본값(`APPENDIX_A`)을 사용하여 계산한 경우, 해당 항목과 적용된 수치를 리포트에 별도로 표기하여 설계자가 인지할 수 있게 해야 한다.

## 5. 보안 및 데이터 보호 (A10)

공학 설계 데이터의 무결성과 기밀성을 보장하기 위해 다음 원칙을 준수합니다.

### 5.1 계층적 권한 분리 (RBAC)
시스템의 라이브러리 데이터는 다음의 권한 체계에 따라 보호됩니다.
*   **Standard Library (Read-Only):** `LibraryManager`에 내장된 표준 배관 규격, 재질 물성, 피팅 계수는 사용자가 직접 수정할 수 없으며 시스템에 의해 보호됩니다.
*   **User Extensions (Read-Write):** 사용자가 추가한 커스텀 기기나 특수 재질은 사용자 프로젝트 범위 내의 별도 파일로 관리되며 자유로운 수정이 허용됩니다.

### 5.2 [S-NFR-001] 파일 무결성 및 원자적 저장 (Atomic Save)
*   **원자적 저장:** 데이터 저장 시 중간 실패로 인한 파일 손상을 막기 위해 임시 파일에 먼저 쓰고 성공 시 원본을 덮어쓰는(Atomic) 방식을 강제합니다.
*   **프로젝트 체크섬:** 저장 시 메타데이터에 체크섬(SHA-256)을 포함하여 파일 오염 여부를 감지합니다.
*   **로컬 처리 원칙:** 모든 설계 데이터는 로컬 환경 내에서만 처리되며 외부로 자동 전송되지 않습니다.
