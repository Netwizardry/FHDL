# FHDL 프로그램 명세서 6차 DSL 및 토폴로지 감사 보고서 (v6.0)

## 1. 감사 개요
- **성격:** DSL 문법의 실제 구현 및 그래프 토폴로지 정합성 감사
- **주안점:** 포트(In/Out) 명확성, 가상 노드 주입, 물성 전파, 루프 검출
- **일자:** 2026-04-02

---

## 2. 치명적 결함 목록 (Critical Findings)

### [C01] 컴포넌트 입출력 포트(Port) 정의 누락 (CRT)
- **현상:** 펌프, 탱크 등 방향성이 중요한 기기에서 Inlet/Outlet 포트가 문법적으로 구분되지 않음.
- **영향:** 사용자가 배관을 반대로 연결해도 에러 없이 계산이 진행되어 물리적으로 불가능한 결과(역류 등)를 초래함.
- **보정권고:** DSL에 포트 명시 문법(예: `pump1.in`, `pump1.out`)을 도입하거나, 기기 타입별로 연결 방향성을 강제하는 검증 로직 추가.

### [C02] 엣지-노드 압력 매핑 로직 불일치 (CRT)
- **현상:** 배관(Pipe)은 엣지인데, 명세상으로는 압력 손실이 배관 중간에서 발생하는지 끝점 노드에서 발생하는지 모호함.
- **영향:** 경로가 복잡해질 때 손실수두가 중복 계산되거나 누락되어 최불리 경로 양정 산출이 부정확해짐.
- **보정권고:** `Network Layer`에서 모든 `Pipe` 엣지를 `[StartNode] -> [FrictionEdge] -> [EndNode]` 구조로 정규화하고, 노드와 엣지의 책임 경계(Static Head vs Dynamic Loss)를 확정할 것.

## 3. 중대 결함 목록 (Major Findings)

### [M01] 동적 물성 전파(Property Flow) 명세 부재 (MAJ)
- **현상:** 전역 온도가 모든 컴포넌트에 고정 적용됨.
- **영향:** 향후 온도 변화가 있는 시스템(냉각탑 등)으로 확장 시 아키텍처를 전면 재수정해야 함.
- **보정권고:** `Calculation Layer`에 `Context Propagation` 엔진을 도입하여 상류의 물성 정보가 하류로 흐르도록 명세 보완.

### [M02] 토폴로지 루프(Loop) 처리 및 검증 미비 (MAJ)
- **현상:** 단순 트리 구조 위주의 명세로 인해, 의도치 않은 폐회로 구성 시 솔버의 무한 루프 가능성 존재.
- **보정권고:** `Network Builder` 단계에서 `Cycle Detection`을 강제하고, 허용된 루프(병렬 배관)에 대한 수렴 조건 명시.

## 4. 최종 판정 (Re-evaluated)
- **판정:** ❌ **부적합 (Fail)**
- **이유:** 문법이 "텍스트"로서는 예쁘지만, "수리 해석을 위한 그래프"로 변환되는 과정에서 포트 개념과 물리적 계산 경계가 모호함. 이는 실제 복잡한 관망 설계 시 반드시 수치 오류로 이어질 구조적 결함임.
