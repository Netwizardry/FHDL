# 04. UI/UX 요구사항 (UI/UX Requirements)

FHDL 시스템의 사용자 인터페이스는 설계자가 설비를 기술하고 계산 결과를 검증하여 최종 사양을 도출하는 전 과정을 효율적으로 지원해야 합니다.

## 1. UI/UX 기본 원칙

*   **입력과 결과의 논리적 분리:** 사용자가 입력한 설계 의도(DSL)와 시스템의 계산 결과(Table/Summary)를 명확히 구분하여 인지적 혼동을 방지합니다.
*   **구조 가시성 (Topology Visibility):** 텍스트 선언이 실제 배관 네트워크로 어떻게 연결되었는지 시각적으로 검토할 수 있어야 합니다.
*   **데이터 성격의 구분:** 사용자가 직접 지정한 값(`manual`), 시스템이 선정한 값(`auto`), 엔지니어링 기본값(`default`)을 시각적으로 차별화합니다.
*   **오류 우선 및 이동성:** 문법/수리적 오류를 최우선으로 표시하며, 오류 메시지 클릭 시 해당 원문 위치로 즉시 이동할 수 있어야 합니다.
*   **반복 설계 최적화:** '수정 - 재계산 - 결과 비교'의 주기를 최소화하도록 빠른 실행 흐름을 제공합니다.

## 2. 5대 핵심 화면 구성

### 2.1 프로젝트 관리 화면 (Project Selection)
*   **기능:** 새 설계 생성, 기존 프로젝트 열기, 최근 작업 목록 및 샘플 불러오기.
*   **목적:** 설계를 파일 및 프로젝트 단위로 관리하는 시작점 제공.

### 2.2 FHDL 에디터 화면 (Input Editor)
*   **기능:** 줄번호, 구문 강조(Syntax Highlighting), 자동 들여쓰기, 실시간 린팅(Linter).
*   **목적:** 설계자가 DSL 문법을 빠르고 정확하게 입력할 수 있는 환경 제공.

### 2.3 구조 및 네트워크 확인 화면 (Topology Viewer)
*   **기능:** 설비 구성요소 리스트, 상류/하류 연결 관계 트리, 최불리 경로 강조.
*   **목적:** 선언된 위상의 논리적 단절이나 연결 오류를 시각적으로 검토.

### 2.4 계산 결과 대시보드 (Result Dashboard)
*   **기능:** 
    *   **요약:** 총 유량, 총 요구양정, 권장 펌프/탱크 사양.
    *   **상세:** 구간별 유량, 관경, 유속, 손실수두를 포함한 데이터 테이블.
*   **목적:** 설계 목표 달성 여부를 최종적으로 판정하고 사양을 확정.

### 2.5 에러 및 경고 센터 (Error/Warning Center)
*   **기능:** 문법 오류, 수리적 부적합(과속, 저압 등), 계산 불능 원인 통합 표시.
*   **목적:** 문제 해결을 위한 구체적인 가이드와 수정 경로 제공.

## 3. 사용자 작업 흐름 (User Workflow)

1.  **시작:** 프로젝트 생성 또는 기존 파일 로드.
2.  **기술:** FHDL 문법으로 설비 구성 및 제약조건 입력.
3.  **검증:** 실시간 문법 검사 및 구조 확인 화면에서 위상 검토.
4.  **해석:** 계산 실행(`Run`) 및 결과 대시보드 검토.
5.  **조정:** 경고 발생 시 에디터로 돌아가 관경 또는 펌프 사양 수정 후 재계산.
6.  **확정:** 설계 결과서 출력 및 프로젝트 저장.
