# 04. UI/UX 요구사항 (UI/UX Requirements)

**Status:** Active | **Version:** v4.0 | **Last Revised:** 2026-04-02 | **Supersedes:** v1.5


FHDL 시스템의 사용자 인터페이스는 설계자가 설비를 기술하고 계산 결과를 검증하여 최종 사양을 도출하는 전 과정을 효율적으로 지원해야 합니다.

## 1. UI/UX 기본 원칙

*   **입력과 결과의 논리적 분리:** 사용자가 입력한 설계 의도(DSL)와 시스템의 계산 결과(Table/Summary)를 명확히 구분하여 인지적 혼동을 방지합니다.
*   **구조 가시성 (Topology Visibility):** 텍스트 선언이 실제 배관 네트워크로 어떻게 연결되었는지 시각적으로 검토할 수 있어야 합니다.
*   **데이터 성격의 구분:** 사용자가 직접 지정한 값(`manual`), 시스템이 선정한 값(`auto`), 엔지니어링 기본값(`default`)을 시각적으로 차별화합니다.
*   **오류 우선 및 이동성:** 문법/수리적 오류를 최우선으로 표시하며, 오류 메시지 클릭 시 해당 원문 위치로 즉시 이동할 수 있어야 합니다.
*   **반복 설계 최적화:** '수정 - 재계산 - 결과 비교'의 주기를 최소화하도록 빠른 실행 흐름을 제공합니다.

## 2. 5대 핵심 화면 구조 및 책임 (Standard Layout)

FHDL GUI는 다음 5개 핵심 패널로 구성되며, 각 패널은 독립적인 책임을 가지고 상호작용해야 한다. 이 구조는 `14_GUI.md` 구현의 절대적 기준이 된다.

### 2.1 [R-UI-001] 프로젝트 관리 패널 (Project Selection)
*   **책임:** 파일 I/O 및 프로젝트 컨텍스트 관리.
*   **기능:** 신규 프로젝트 생성, `.fhproj` 파일 로드, 최근 작업 목록(LRU) 관리.
*   **상호작용:** 프로젝트 로드 시 `Editor`와 `Topology Viewer`에 데이터를 공급한다.

### 2.2 [R-UI-002] FHDL 에디터 패널 (Input Editor)
*   **책임:** DSL 소스 코드 편집 및 실시간 문법 피드백.
*   **기능:** Syntax Highlighting, 줄번호 표시, 자동 들여쓰기, 실시간 린팅(Debounced).
*   **상호작용:** 코드 변경 시 `Topology Viewer`를 즉시 갱신하고, `Error Center`에 문법 오류를 전달한다.

### 2.3 [R-UI-003] 네트워크 토폴로지 뷰어 (Topology Viewer)
*   **책임:** 선언된 위상의 시각적 검증 및 논리적 구조 표시.
*   **기능:** 노드-엣지 다이어그램, 상류/하류 연결 트리, 최불리 경로(Worst-case) 하이라이트.
*   **상호작용:** 노드 클릭 시 `Editor`의 해당 라인으로 포커스를 이동시키고, `Result Dashboard`의 해당 행을 강조한다.

### 2.4 [R-UI-004] 계산 결과 대시보드 (Result Dashboard)
*   **책임:** 수리 해석 결과 및 자동 산정 결과의 정량적 제시.
*   **기능:** 요약 정보(총유량/양정), 상세 데이터 테이블(관경, 유속, 손실), 선정 근거(Provenance) 툴팁.
*   **상호작용:** `manual` 지정 값 변경 시 즉시 재계산 트리거를 제안한다.

### 2.5 [R-UI-005] 에러 및 경고 센터 (Error/Warning Center)
*   **책임:** 시스템 내 모든 진단 메시지의 통합 관리 및 가이드 제공.
*   **기능:** 오류 등급별 필터링, 진단 코드(`SYN001` 등) 표시, 해결 방법(Remedy) 제안.
*   **상호작용:** 오류 항목 더블클릭 시 `Editor`의 발생 위치로 강제 이동(Jump to Source)한다.

## 3. 사용자 작업 흐름 (User Workflow)

1.  **시작:** 프로젝트 생성 또는 기존 파일 로드.
2.  **기술:** FHDL 문법으로 설비 구성 및 제약조건 입력.
3.  **검증:** 실시간 문법 검사 및 구조 확인 화면에서 위상 검토.
4.  **해석:** 계산 실행(`Run`) 및 결과 대시보드 검토.
5.  **조정:** 경고 발생 시 에디터로 돌아가 관경 또는 펌프 사양 수정 후 재계산.
6.  **확정:** 설계 결과서 출력 및 프로젝트 저장.
