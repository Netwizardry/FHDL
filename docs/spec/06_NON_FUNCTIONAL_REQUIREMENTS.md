# 06. 비기능 요구사항 (Non-Functional Requirements)

FHDL 시스템은 정확한 수치 계산뿐만 아니라, 엔지니어링 설계 도구로서의 안정성과 신뢰성을 보장하기 위해 다음과 같은 비기능적 품질 기준을 준수해야 합니다.

## 1. 장애 허용성 (Fault Tolerance)

일부 모듈의 오류나 입력 데이터의 결함이 시스템 전체의 붕괴로 이어지지 않도록 합니다.
*   **장애 격리 (Isolation):** 문법 오류가 발생하더라도 텍스트 에디터와 오류 목록 표시 기능은 정상적으로 유지되어야 합니다.
*   **상태 보존:** 계산 파이프라인 중단 시에도 현재까지 편집된 문서 내용과 이전의 유효한 진단 정보가 소실되지 않아야 합니다.
*   **부분 성공 허용:** `Warning` 수준의 비치명적 결함이 있는 경우에도, 시스템은 가능한 범위 내에서 계산을 수행하고 결과를 제시해야 합니다.

## 2. 안전 우선성 (Fail-Safe)

불확실한 상황에서 위험한 자동 결정을 내리기보다 보수적으로 멈추거나 경고하는 것을 원칙으로 합니다.
*   **보수적 산정:** 계산 근거(필수 입력 등)가 부족할 경우 억측에 의한 자동 산정을 수행하지 않고 즉시 계산 불능 사유를 고지합니다.
*   **임의 선택 금지:** 허용 범위를 만족하는 표준 관경이 없을 경우, 임의의 관경을 추천하지 않고 `CAL005` 오류를 발생시켜 설계자의 개입을 유도합니다.
*   **제약 준수 강제:** 사용자가 직접 지정한 값(`manual`)이 설계 제약조건(과속 등)을 위반할 경우, 이를 확정 결과가 아닌 '부적합' 상태로 명시합니다.

## 3. 설명 가능성 (Explainability)

시스템이 도출한 모든 자동 산정 결과에 대해 설계자가 그 근거를 추적하고 검토할 수 있어야 합니다.
*   **근거 연결 (Traceability):** 특정 관경이 선정된 이유(예: 유속 1.8m/s 만족), 총 요구양정이 결정된 이유(예: 최불리 경로 n8번 수두 합산)를 명확히 제시합니다.
*   **계산 분해 (Breakdown):** 최종 수치만 보여주지 않고, `마찰손실 + 정수두 + 말단요구압`과 같은 중간 계산 과정의 분해 내역을 제공합니다.
*   **기본값 명시:** 시스템이 엔지니어링 표준 기본값을 사용하여 계산한 경우, 해당 항목과 적용된 수치를 리포트에 별도로 표기하여 설계자가 인지할 수 있게 합니다.
