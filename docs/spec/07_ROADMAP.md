# 16. 구현 우선순위 및 MVP 범위 명세

본 문서는 FHDL 시스템의 단계별 개발 계획을 수립하고, 초기 버전(MVP)에서 집중해야 할 핵심 기능의 경계를 정의하여 프로젝트의 성공적인 착수를 돕습니다.

## 1. 구현 기본 원칙

*   **코어 우선 (Core First):** 화려한 UI보다는 파서, 계산 엔진, 자동 선정 로직 등 시스템의 '두뇌'를 최우선으로 개발합니다.
*   **단계적 완성:** 각 단계는 '동작 가능한 최소 단위'가 되어야 합니다. 파서만 있는 상태보다 '파서+최소계산'이 연결된 루프를 지향합니다.
*   **리스크 통제:** 복잡한 루프망 해석이나 실시간 시각화 등 실패 가능성이 높은 고난도 기능은 후순위(MVP 이후)로 배치합니다.
*   **검증 가능성:** 모든 구현은 정의된 테스트 시나리오(T-CAL, T-AUTO 등)를 통해 즉시 검증 가능해야 합니다.

## 2. MVP(Minimum Viable Product) 정의

FHDL MVP는 **"실무용 1차 수리 계산기 및 설계 검토기"**로 정의하며, 아래 기능을 최소 포함합니다.

### 2.1 MVP 포함 범위 (In-Scope)
1.  **입력:** 텍스트 기반 DSL 입력, 기본 키워드(system, pump, pipe 등) 및 단위 파싱 지원.
2.  **구조:** 단일 공급원 기반의 트리형 및 단순 병렬 배관망 처리.
3.  **계산:** 총유량 집계, 구간별 유속/손실 계산, 최불리 경로 탐색 및 총 요구양정 산출.
4.  **산정:** 표준 관경 테이블 기반 관경 자동 선정, 펌프 및 탱크 사양 제안.
5.  **출력:** CLI 또는 단순 GUI를 통한 요약 리포트, 구간별 상세표, 진단(오류/경고) 메시지.

### 2.2 MVP 제외 범위 (Out-of-Scope)
*   복잡한 루프망(Mesh) 유량 분배 해석 및 다중 공급원 시스템.
*   온도/점도 변화의 정밀 반영 및 NPSH/캐비테이션 상세 검토.
*   드래그 앤 드롭 방식의 그래픽 편집기 및 실시간 3D 시각화.
*   기자재 카탈로그 DB 연동 및 CAD 도면 자동 생성.

## 3. 8단계 구현 로드맵 (Phases)

| 단계 | 명칭 | 핵심 목표 |
| :--- | :--- | :--- |
| **Phase 1** | Parser | DSL 텍스트를 읽고 구문 트리(AST) 생성 및 문법 오류 검출 |
| **Phase 2** | Semantic | 단위 변환, 참조 해결 및 의미론적 정합성 검증 |
| **Phase 3** | Network | 엔티티를 계산용 노드-엣지 그래프로 변환 |
| **Phase 4** | Core Calc | 유량 집계, 손실 및 양정의 핵심 물리 계산 엔진 가동 |
| **Phase 5** | Auto Sizing | 계산 결과와 제약조건을 결합하여 관경/펌프/탱크 최적 선정 |
| **Phase 6** | Diagnostics | 리포트 포맷팅 및 설명 가능성(Explainability) 근거 출력 |
| **Phase 7** | Interface | CLI 또는 단순 GUI를 통한 사용자 작업 루프 완성 |
| **Phase 8** | Expansion | 규칙 테이블 외부화, 결과 비교 등 고도화 기능 추가 |

## 4. MVP 완료 기준 (Definition of Done)

*   **기능:** 정상 FHDL 문서 파싱부터 수리 계산, 자동 선정, 리포트 출력까지의 엔드투엔드 흐름 성공.
*   **품질:** MVP 필수 테스트셋(T-SYN, T-CAL, T-AUTO) 100% 통과.
*   **안정성:** 치명적 오류 발생 시 비정상 종료 없이 진단 메시지를 정확히 출력.
